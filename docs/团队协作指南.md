## 1. 准备工作

### 1.1 安装 Git

**Windows 用户：**
1. 访问 https://git-scm.com/downloads
2. 下载 Windows 版本
3. 双击安装，全部默认选项即可
4. 安装完成后，右键桌面，看到 "Git Bash Here" 表示安装成功

**Mac 用户：**
```bash
# 打开终端，输入以下命令
xcode-select --install
```

**验证安装：**
```bash
git --version
# 输出类似：git version 2.39.2 表示成功
```

### 1.2 注册 Gitee 账号

1. 访问 https://gitee.com/signup
2. 填写手机号/邮箱，完成注册
3. 记住你的 **用户名** 和 **邮箱**

### 1.3 配置 Git 用户信息

打开终端（Windows 打开 Git Bash），输入：

```bash
# 替换成你自己的名字和邮箱
git config --global user.name "你的名字"
git config --global user.email "你的邮箱@example.com"

# 验证配置
git config --global user.name
git config --global user.email
```

### 1.4 配置 SSH 密钥（推荐）

SSH 密钥可以让你不用每次都输入密码。

**第一步：生成 SSH 密钥**
```bash
# 一路回车，使用默认设置
ssh-keygen -t rsa -C "你的邮箱@example.com"
```

**第二步：查看公钥**
```bash
# Mac/Linux
cat ~/.ssh/id_rsa.pub

# Windows
type %USERPROFILE%\.ssh\id_rsa.pub
```

**第三步：添加到 Gitee**
1. 复制上面显示的公钥（整行内容）
2. 登录 Gitee → 点击头像 → 设置 → SSH公钥
3. 粘贴公钥，点击确定

**第四步：测试连接**
```bash
ssh -T git@gitee.com
# 显示 "Hi 用户名! You've successfully authenticated" 表示成功
```

## 3. 日常开发流程

### 3.1 标准工作流程图

```
开始
  ↓
拉取最新代码 (git pull)
  ↓
创建/切换分支 (git checkout)
  ↓
开发功能
  ↓
保存更改 (git add + git commit)
  ↓
推送分支 (git push)
  ↓
通知组长合并
  ↓
结束
```

### 3.2 每天开始工作前

```bash
# 1. 进入项目目录
cd ~/Projects/crems

# 2. 切换到 master 分支
git checkout master

# 3. 拉取最新代码
git pull origin master
```

### 3.3 开始新功能开发

```bash
# 1. 从 master 创建新分支
git checkout -b feature/你的功能名称

# 例如：
git checkout -b feature/student-search
git checkout -b feature/company-profile
git checkout -b feature/zhangsan-dev
```

### 3.4 开发过程中保存代码

```bash
# 查看你修改了哪些文件
git status

# 添加所有修改的文件
git add .

# 或者只添加特定文件
git add 文件路径

# 提交更改
git commit -m "feat: 添加了xxx功能"
```

### 3.5 推送到远程仓库

```bash
# 第一次推送新分支
git push origin feature/你的功能名称

# 之后的推送
git push
```

## 4. 提交代码详解

### 4.1 查看文件状态

```bash
git status
```

**常见状态说明：**
| 状态 | 含义 |
|------|------|
| `Untracked files` | 新文件，Git 还没跟踪 |
| `Changes not staged for commit` | 已修改，但还没暂存 |
| `Changes to be committed` | 已暂存，等待提交 |
| `nothing to commit, working tree clean` | 没有修改，工作区干净 |

### 4.2 暂存文件（git add）

```bash
# 添加指定文件
git add src/main/java/MyClass.java

# 添加所有修改的文件
git add .

# 添加某个文件夹下的所有修改
git add backend/
```

### 4.3 提交更改（git commit）

```bash
# 提交并添加说明
git commit -m "feat: 添加学生搜索功能"

# 如果需要更详细的说明
git commit -m "feat: 添加学生搜索功能

- 实现按姓名搜索
- 实现按专业搜索
- 添加分页支持"
```

### 4.4 查看提交历史

```bash
# 查看简洁的历史记录
git log --oneline

# 查看详细的提交记录
git log

# 查看某个文件的修改历史
git log --follow 文件路径
```

## 5. 同步最新代码

### 5.1 为什么需要同步？

当其他人提交了新代码，你的本地仓库不是最新的，需要同步后才能避免冲突。

### 5.2 同步 master 分支

```bash
# 切换到 master
git checkout master

# 拉取最新代码
git pull origin master
```

### 5.3 同步你的开发分支

```bash
# 切换到你的分支
git checkout feature/你的功能名称

# 合并 master 的最新代码到你的分支
git merge master

# 解决冲突（如果有的话）
# ... 解决后 ...
git add .
git commit -m "merge: 合并 master 最新代码"
```

### 5.4 使用 rebase 保持历史整洁（推荐）

```bash
# 切换到你的分支
git checkout feature/你的功能名称

# 变基到最新的 master
git rebase master

# 如果有冲突，解决后继续
git add .
git rebase --continue
```

## 6. 分支管理

### 6.1 分支命名规范

| 分支类型 | 命名格式 | 示例 |
|---------|---------|------|
| 主分支 | `master` | master |
| 功能分支 | `feature/功能名` | feature/student-list |
| 修复分支 | `fix/问题描述` | fix/login-error |
| 个人开发分支 | `dev/姓名` | dev/zhangsan |

### 6.2 常用分支命令

```bash
# 查看所有分支
git branch -a

# 创建新分支
git branch feature/new-feature

# 切换分支
git checkout feature/new-feature

# 创建并切换到新分支
git checkout -b feature/new-feature

# 删除本地分支（已合并的）
git branch -d feature/old-feature

# 强制删除分支
git branch -D feature/old-feature
```

### 6.3 分支使用建议

```
master (主分支，保持稳定)
  │
  ├── feature/zhangsan-功能A (张三的功能A分支)
  ├── feature/lisi-功能B (李四的功能B分支)
  └── feature/wangwu-功能C (王五的功能C分支)
```

**原则：**
- 不要直接在 master 上开发
- 每个功能创建一个分支
- 开发完成后合并回 master
- 合并前先同步最新代码

## 7. 常见问题解决

### 7.1 提交时提示 "Please tell me who you are"

**原因：** 没有配置 Git 用户信息

**解决：**
```bash
git config --global user.name "你的名字"
git config --global user.email "你的邮箱@example.com"
```

### 7.2 推送时提示 "Permission denied"

**原因：** 没有权限或 SSH 配置问题

**解决：**
1. 确认你已经被添加为仓库成员
2. 检查 SSH 密钥是否配置正确
3. 或者使用 HTTPS 方式推送

### 7.3 出合并冲突了怎么办？

**什么是冲突？**
当两个人修改了同一个文件的同一个位置，Git 无法自动合并，需要手动解决。

**解决步骤：**

**第一步：查看冲突文件**
```bash
git status
# 会显示 "both modified: 文件名"
```

**第二步：打开冲突文件**
文件中会有这样的标记：
```
<<<<<<< HEAD
你的修改内容
=======
别人的修改内容
>>>>>>> commit-id
```

**第三步：手动解决**
- 删除 `<<<<<<<`、`=======`、`>>>>>>>` 这些标记
- 保留你想要的内容
- 保存文件

**第四步：提交解决结果**
```bash
git add 冲突文件
git commit -m "fix: 解决合并冲突"
```

### 7.4 不小心提交了错误的代码

**情况1：还没推送到远程**
```bash
# 撤销最后一次提交，保留修改
git reset --soft HEAD~1

# 重新修改后再次提交
git add .
git commit -m "正确的提交信息"
```

**情况2：已经推送到远程**
```bash
# 创建一个新的提交来撤销之前的修改
git revert HEAD
git push
```

### 7.5 想丢弃本地所有修改

```bash
# 丢弃所有未提交的修改（危险操作！）
git checkout -- .

# 丢弃某个文件的修改
git checkout -- 文件路径
```

### 7.6 拉取代码时提示冲突

```bash
# 暂存本地修改
git stash

# 拉取最新代码
git pull origin master

# 恢复本地修改
git stash pop

# 解决可能的冲突
```

## 8. 提交规范

### 8.1 提交信息格式

```
<类型>: <简短描述>

<详细说明（可选）>
```

### 8.2 类型说明

| 类型 | 说明 | 示例 |
|------|------|------|
| `feat` | 新功能 | feat: 添加学生搜索功能 |
| `fix` | 修复 bug | fix: 修复登录失败问题 |
| `docs` | 文档更新 | docs: 更新 README |
| `style` | 代码格式调整 | style: 格式化代码 |
| `refactor` | 重构代码 | refactor: 重构用户模块 |
| `test` | 添加测试 | test: 添加单元测试 |
| `chore` | 构建/工具变更 | chore: 更新依赖 |

### 8.3 提交示例

```bash
# 好的提交信息
git commit -m "feat: 添加按专业筛选学生功能"
git commit -m "fix: 修复企业信息保存失败的问题"
git commit -m "docs: 添加 API 接口文档"

# 不好的提交信息（避免这样写）
git commit -m "修改代码"
git commit -m "update"
git commit -m "fix bug"
```

### 8.4 提交频率建议

- ✅ 完成一个小功能就提交
- ✅ 修复一个 bug 就提交
- ❌ 攒一大堆修改最后一次性提交
- ❌ 提交半成品代码
